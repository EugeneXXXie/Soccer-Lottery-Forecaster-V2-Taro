# SoccerLotteryForecasterTaro

一个基于 **Taro 4 + Vue 3 + TypeScript + NutUI** 重写的总进球数投注辅助项目，面向 H5 优先，同时保留 Taro 多端扩展能力。

当前版本重点解决了以下问题：
- 将旧版 Web 项目迁移为 Taro 项目结构
- 将算法逻辑与页面展示逻辑解耦
- 支持桌面端与移动端差异化交互
- 提供 **稳健 / 激进 / 冲刺** 三档投注策略
- 根据赔率与相对概率，尽量扩大可盈利结果覆盖范围

## 快速概览

- **部署方式**：本地 Node.js 环境部署，默认以 H5 方式开发与运行
- **运行命令**：`npm run dev:h5` 或 `cnpm run dev:h5`
- **构建命令**：`npm run build:h5`
- **技术栈**：Taro 4、Vue 3、TypeScript、Sass、NutUI
- **数据来源**：`sporttery.cn` 体彩总进球数相关接口
- **开源协议**：MIT License

## 功能特性

### 1. 比赛选择
- 支持从接口拉取体彩赛事数据
- 支持自定义赔率模式
- 桌面端采用弹层选择，移动端保留 picker 交互

### 2. 赔率输入
- 支持 0 球到 7+ 球共 8 档赔率输入
- 支持锁定 / 解锁赔率输入
- 输入错误时给出单项提示

### 3. 预算设置
- 最低预算限制为 **16 元**
- 自动按 **2 元 / 股** 进行预算归一化
- 自动修正为偶数金额

### 4. 策略切换
- **稳健**：更偏向保护低赔率结果
- **激进**：兼顾低赔率保护与覆盖扩张
- **冲刺**：更愿意把预算推向高回报区间

### 5. 结果展示
- 展示每个进球数档位的：
  - 预期投入股数
  - 投入金额
  - 预期回报
- 根据回报与总预算关系动态着色
- 汇总当前策略、实际预算、盈利档数与建议结论

## 算法解析

当前算法不是简单按赔率平均分配，而是围绕下面几个目标展开：

1. **给所有进球数档位保留基础持仓**
2. **优先保护低赔率结果**
3. **尽量让更多档位达到回本或盈利线**
4. **根据策略风格处理剩余仓位**

### 1. 输入数据

算法接收两类输入：

- 8 个总进球赔率
- 用户预算金额

对应关系：

- `s0` → 0球
- `s1` → 1球
- `s2` → 2球
- `s3` → 3球
- `s4` → 4球
- `s5` → 5球
- `s6` → 6球
- `s7` → 7+球

### 2. 预算归一化

由于本项目按 **2 元 / 股** 计算，因此预算会先进行归一化：

- 小于 16 元视为无效
- 非整数向下取整
- 奇数金额自动减 1，修正为偶数

例如：

- `51` → `50`
- `80` → `80`
- `15` → 无效

### 3. 相对概率估算

算法会对赔率做倒数处理，用来近似表示相对概率：

```text
probabilityWeight ≈ 1 / odds^power
```

含义是：

- 赔率越低，通常代表相对概率越高
- 赔率越高，通常代表相对概率越低

不同策略会使用不同的 `power`，从而体现不同的保守 / 激进程度。

### 4. 盈利线计算

算法会先计算每个结果要达到“回本 / 盈利线”至少需要多少股：

```text
breakEvenShares = ceil(principal / (odds × 2))
```

例如：

- 总预算 80 元
- 某档赔率 4.0

则该档位理论上至少需要：

```text
80 / (4.0 × 2) = 10 股
```

也就是说，该结果命中时，预期回报才更容易覆盖总预算。

### 5. 低赔率保护

算法会优先选择前若干个低赔率档位进行保护。

原因是：

- 低赔率通常意味着相对更可能发生
- 这些档位更值得优先保住
- 这一步能让策略更贴近真实投注直觉，而不是机械均分

不同策略保护数量不同：

- **稳健**：保护更多低赔率结果
- **激进**：保护中等数量低赔率结果
- **冲刺**：只保留少量低赔率保护，把更多空间给高回报区

### 6. 盈利覆盖扩张

在完成低赔率保护后，算法会在剩余预算范围内继续选择：

- 更容易补到盈利线的结果
- 能增加整体盈利档数的结果
- 同时兼顾相对概率与赔率结构

这一步的重点不是“平均分”，而是：

**尽量让更多档位达到可盈利状态。**

### 7. 剩余仓位微调

当低赔率保护与盈利覆盖完成后，如果还有剩余股数，会根据当前策略继续微调：

#### 稳健
- 优先继续补低赔率结果
- 更强调保护相对高概率区间

#### 激进
- 同时考虑低赔率结果和接近盈利线的结果
- 试图在覆盖数量与概率保护之间取得平衡

#### 冲刺
- 更愿意把剩余仓位推向高回报方向
- 允许更高集中度
- 更适合想拉高收益上限的场景

### 8. 输出结果

算法最终输出每个进球档位的：

- `ExpectedInput`：预期投入股数
- `AmountInvested`：投入金额
- `ExpectedReturn`：预期回报

页面会进一步计算：

- 实际预算
- 盈利档数
- 推荐结论

### 9. 推荐文案逻辑

当前推荐文案根据“盈利档数”大致分层：

- 盈利档数较多 → 更值得关注
- 盈利档数中等 → 可结合临场判断
- 盈利档数较少 → 适合谨慎投入
- 无明显盈利覆盖 → 建议谨慎对待

## 数据来源说明

当前比赛数据与赔率信息主要来自体彩相关接口，示例来源：

```text
https://webapi.sporttery.cn/gateway/jc/football/getMatchCalculatorV1.qry?poolCode=ttg&channel=c
```

页面中展示的联赛简称、联赛全称、球队简称等信息，会以接口实际返回字段为准进行映射与展示。
